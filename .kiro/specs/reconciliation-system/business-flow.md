# 委外加工对账系统 — 业务流程全景图

## 一、系统定位

对账系统是委外加工业务链的**结算核心**，衔接上游的"询价→报价→下单→生产→交付"流程与下游的"付款→存档"流程。其核心职责是：确保每一笔付款都有据可查、金额准确、双方确认。同时，系统通过**虚拟入库**机制跟踪因生产异常导致的补发货物，确保订单实际变化有完整记录。

    ▼                   ▼

    │            补发零件    不补发/扣款

    │               │              │

│                                       │

### 3.5 虚拟入库流程

```
补发请求（Re_Shipment）创建
        │
        ▼
┌─────────────────────────────────┐
│ 判断处理方式                     │
└─────────┬───────────────────────┘
          │
    ┌─────┴──────────────────┐
    ▼                        ▼
 实际补发                  不补发（协商决定）
    │                        │
    ▼                        ▼
┌──────────────────┐   ┌──────────────────┐
│ 创建虚拟入库记录  │   │ 创建扣款记录      │
│ Virtual_Inbound  │   │ Deduction        │
├──────────────────┤   ├──────────────────┤
│ • 关联原始工单    │   │ • 关联原始工单    │
│ • 关联异常单号    │   │ • 扣款金额       │
│ • 入库零件/数量   │   │ • 不补发原因     │
│ • 入库类型=虚拟   │   │ • 从订单总额扣除  │
│ • 异常原因备注    │   └────────┬─────────┘
│ • 补发原因备注    │            │
│ • 责任方标识     │            │
└────────┬─────────┘            │
         │                      │
         ▼                      ▼
┌─────────────────────────────────────┐
│ 纳入订单结算明细 (Settlement_Detail) │
├─────────────────────────────────────┤
│ 虚拟入库 → 补发费用行项             │
│ 扣款     → 扣款行项（负数）          │
│ 均影响净利润计算                     │
└─────────────────────────────────────┘
```

**虚拟入库的业务含义：**

- 当因材料方或加工方的问题导致需要补发货物时，补发的内容不经过实际仓库收发流程，而是在系统中创建一条"虚拟入库"记录
- 虚拟入库记录必须标注：异常原因（为什么需要补发）、补发原因（材料方责任/加工方责任）、关联的异常单号
- 如果协商后决定不补发，则直接从最终订单金额中扣除对应款项，并备注不补发的原因
- 无论补发还是扣款，都会反映在订单的结算明细中，影响最终净利润计算

### 3.4 订单结算明细流程

```
委外工单交付且质检合格
        │
        ▼
┌─────────────────────────────────────┐
│ 自动生成 Settlement_Detail           │
├─────────────────────────────────────┤
│ 收入项：                             │
│   • 客户付款金额                     │
├─────────────────────────────────────┤
│ 成本项：                             │
│   • 各工序加工费（逐项）              │
│   • 物流费用                         │
│   • 补发货物费用（来自虚拟入库记录）   │
│   • 扣款金额（不补发时从订单扣除）     │
│   • 重新安排加工费用                  │
├─────────────────────────────────────┤
│ 虚拟入库关联：                        │
│   • 自动拉取该订单的 Virtual_Inbound  │
│   • 展示补发原因 + 异常备注           │
│   • 补发费用计入成本                  │
├─────────────────────────────────────┤
│ 净利润 = 客户付款 - 成本合计          │
└─────────────┬───────────────────────┘
              │
              ▼
┌─────────────────────┐
│ draft 状态：可编辑   │  财务人员增删改行项
└─────────┬───────────┘
          │ 确认
          ▼
┌─────────────────────┐
│ finalized 状态       │  关联至对账单
│ 附件不可变           │  可生成 PDF 结算单
└─────────────────────┘
```

## 四、角色与权限矩阵

| 角色 | 核心操作 |
|------|---------|
| **财务人员** | 生成对账单、编辑行项、提出调整、录入付款、导出报表、上传凭证、确认结算明细 |
| **财务主管** | 审批调整（≤1000元）、查看仪表盘、管理报表 |
| **财务总监** | 审批调整（>1000元）、升级审批 |
| **供应商** | 查看对账单、确认/争议、提交 Supplier_Claim |
| **业务人员** | 创建生产异常、判定责任、选择处理方式 |
| **系统管理员** | 数据回滚（24h内）、权限管理、审计日志查看 |

## 六、关键状态机

### 6.1 对账单状态

```
pending ──→ confirmed ──→ paid
   │              ↑
   │              │ (异常解决后重新确认)
   └──→ disputed ─┘
   │
   └──→ timeout (15天未响应)
```

### 6.2 异常处理状态

```
open → investigating → resolved → closed
```

### 6.3 调整审批状态

```
pending_approval → approved → (金额更新)
       │
       └──→ rejected
       │
       └──→ escalated (3工作日超时自动升级)
```

### 6.4 付款状态

```
pending_payment → partially_paid → paid
       │                              ↑
       └──────────────────────────────┘ (一次付清)
```

### 6.5 结算明细状态

```
draft → finalized (不可逆)
```

### 6.6 虚拟入库状态

```
pending → confirmed → linked_to_settlement
   │
   └──→ cancelled (协商后取消补发，转为扣款)
```

## 七、定时任务与自动触发

| 触发条件 | 动作 |
|---------|------|
| 每月末 | 自动生成当月对账单 |
| 对账单发出后 7 天 | 催办提醒（pending 状态） |
| 对账单发出后 15 天 | 标记超时，通知财务 |
| 审批请求 3 个工作日未处理 | 催办 + 自动升级 |
| 每日 | 更新仪表盘统计数据 |
| 工单交付且质检合格 | 自动生成结算明细 |
| 对账单 confirmed | 自动生成付款申请 |
| 补发请求（Re_Shipment）确认发货 | 自动创建虚拟入库记录（Virtual_Inbound） |
| 协商决定不补发 | 自动创建扣款记录，从订单总额扣除 |

## 九、安全与合规

- **审计日志**：所有操作（创建/修改/确认/审批/导出）全程记录，日志不可删改
- **数据不可变**：confirmed/paid 状态的对账单禁止修改
- **权限控制**：基于角色的访问控制，未授权访问记录安全事件
- **数据回滚**：仅限 24 小时内误操作，需管理员审批
- **数据保留**：所有历史数据保留至少 3 年
- **附件保护**：finalized 状态的结算明细，关联附件不可删除/替换
