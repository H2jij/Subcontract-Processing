# 图纸库管理方案

> 在委外加工系统中建立图纸数据库，存储拆分后的零件子图路径。
> 询价时根据模具号+零件号查找图纸，找不到则从本地盘按路径层级拿到原图自动拆分。
> 拆分结果保存到项目目录并记录到数据库，供加工商下载查看。

---

## 一、核心思路

```
只有一个数据库表（图纸表）
存储：模具号 + 零件号 → 子图文件路径

查找流程：
  模具号 + 零件号
    ↓
  查数据库 → 有 → 返回下载链接
    ↓ 没有
  按路径层级找原图（E:\年份\区间\型号\4.模具图\2D\xxx.dwg）
    ↓ 找到
  拆图 → 保存子图到项目目录 → 写入数据库 → 返回下载链接
    ↓ 找不到
  标记"无图纸"
```

**不需要套图表。** 原图就在本地盘上，按固定目录结构存放，`find_latest_dwg_path()` 函数直接按路径层级查找即可。套图表只是把查找结果多存一份，没有意义。

---

## 二、原图路径层级（参考现有采购系统）

系统部署在图纸所在的本机，原图存放在本地盘（如 E:\），目录结构如下：

```
E:\
└── 2025---M250001~300\                    ← 年份目录（按序号范围分组）
    ├── 2501~250100\                       ← 序号区间目录
    │   └── M250247-P6.2025.04.01\        ← 型号目录（模具号.日期版本）
    │       └── 4.模具图\
    │           └── 2D\                    ← DWG 文件所在
    │               └── M250247-P6.2025.04.01.dwg  ← 套图（原图/大图）
    └── ...
```

查找函数 `find_latest_dwg_path(模具号)` 会自动：
1. 从模具号提取序号（M250247 → 250247）
2. 根据序号前缀匹配年份目录（M25 → 2025---M250001~300）
3. 根据序号匹配区间目录
4. 根据模具号匹配型号目录
5. 定位 `4.模具图\2D\` 子目录
6. 选最新版本的 DWG 文件

## 三、数据库设计

### 唯一一张表：entrust_drawings（图纸表）

存储拆分后的零件子图信息。支持多版本，同一模具+同一零件可保留多个版本的图纸，
用户通过下拉框选择使用哪个版本。

```sql
CREATE TABLE entrust_drawings (
    id              SERIAL PRIMARY KEY,
    mold_code       VARCHAR(64) NOT NULL,            -- 模具编号 如 M250247-P6
    part_code       VARCHAR(64) NOT NULL,            -- 零件编号 如 DIE-10, B03, PH-02
    -- 子图文件信息
    file_name       VARCHAR(255) NOT NULL,           -- 文件名 如 DIE-10_v2.dwg
    file_path       VARCHAR(512) NOT NULL,           -- 项目内相对路径 如 uploads/part_drawings/M250247-P6/DIE-10_v2.dwg
    file_size_kb    INTEGER,                         -- 文件大小(KB)
    -- 版本管理
    version         INTEGER NOT NULL DEFAULT 1,      -- 版本号，每次拆图+1
    is_latest       BOOLEAN NOT NULL DEFAULT TRUE,   -- 是否最新版（新版本产生时旧版本置false）
    -- 来源信息
    source_type     VARCHAR(32) DEFAULT 'auto_split', -- auto_split自动拆分 / manual手动上传
    split_at        TIMESTAMP,                       -- 拆分/上传时间
    -- 状态
    status          VARCHAR(32) DEFAULT 'available',  -- available/unavailable
    remark          TEXT,
    created_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at      TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_drawing_mold_part ON entrust_drawings (mold_code, part_code);
CREATE INDEX idx_drawing_latest ON entrust_drawings (mold_code, part_code, is_latest);
```

**版本管理规则：**
- 同一模具+同一零件允许多条记录（多版本共存）
- 每次重新拆图时，旧记录的 `is_latest` 置为 false，新记录 `version` 自动+1、`is_latest` 为 true
- 文件名带版本号（如 `DIE-10_v2.dwg`），不会互相覆盖
- 询价默认使用最新版，前端下拉框可选历史版本

**示例数据：**

| mold_code | part_code | version | file_name | file_path | is_latest |
|-----------|-----------|---------|-----------|-----------|-----------|
| M250247-P6 | DIE-10 | 1 | DIE-10_v1.dwg | uploads/part_drawings/M250247-P6/DIE-10_v1.dwg | false |
| M250247-P6 | DIE-10 | 2 | DIE-10_v2.dwg | uploads/part_drawings/M250247-P6/DIE-10_v2.dwg | true |
| M250247-P6 | A-10 | 1 | A-10_v1.dwg | uploads/part_drawings/M250247-P6/A-10_v1.dwg | true |
| M250231-P7 | B03 | 1 | B03_v1.dwg | uploads/part_drawings/M250231-P7/B03_v1.dwg | true |
| M250263-P5 | UP | 1 | UP_v1.dwg | uploads/part_drawings/M250263-P5/UP_v1.dwg | true |

## 四、文件存储结构

拆分后的子图按模具号分文件夹，文件名带版本号，存在项目 uploads 目录下：

```
RuoYi-Vue3-FastAPI/
└── uploads/
    ├── drawings/                    # 已有：项目上传的图纸
    └── part_drawings/               # 新增：拆分后的子图文件
        ├── M250247-P6/
        │   ├── DIE-10_v1.dwg       # 旧版本保留
        │   ├── DIE-10_v2.dwg       # 最新版
        │   ├── A-10_v1.dwg
        │   └── B03_v1.dwg
        ├── M250231-P7/
        │   ├── UP_v1.dwg
        │   └── PH-03_v1.dwg
        └── M250263-P5/
            └── A07_v1.dwg
```

- 同一零件多版本共存，文件名带 `_v{version}` 后缀，不会覆盖
- 询价默认用最新版（is_latest=true），前端下拉框可选历史版本

## 五、业务流程

### 5.1 询价时自动查找图纸

```
我方建项目/发起询价，零件信息中有 模具号 + 零件编号
  ↓
发送询价时，后端对每个零件查找图纸：

  ┌──────────────────────────────────────────────┐
  │ Step1: 查数据库                               │
  │   SELECT * FROM entrust_drawings              │
  │   WHERE mold_code='M250247-P6'                │
  │     AND part_code='DIE-10'                    │
  │                                               │
  │   找到 → 拼接下载URL，返回                     │
  │   没找到 ↓                                    │
  │                                               │
  │ Step2: 按路径层级找原图                        │
  │   find_latest_dwg_path('M250247-P6')          │
  │   → E:\2025---M250001~300\...2D\xxx.dwg      │
  │                                               │
  │   找到原图 → 调用拆图引擎                      │
  │     → 拆出 DIE-10 子图                        │
  │     → 旧版本 is_latest 置 false               │
  │     → 保存新版本到 uploads/part_drawings/       │
  │       M250247-P6/DIE-10_v2.dwg                │
  │     → 写入 entrust_drawings（version+1）       │
  │     → 拼接下载URL，返回                        │
  │                                               │
  │   原图也没找到 → 标记"无图纸"                   │
  └──────────────────────────────────────────────┘
  ↓
询价邀请附带图纸下载链接
  ↓
加工商收到 → 零件列表每行有"下载图纸"按钮 → 点击直接下载DWG
我方询价管理页面 → 同样有"下载图纸"按钮
```

### 5.2 拆图执行细节

```
1. find_latest_dwg_path(模具号) → 定位原图 DWG 路径
2. 读取 DWG 文件
3. ODAFileConverter: DWG → DXF（格式转换）
4. ezdxf 解析 DXF，识别所有子图块（按编号）
5. 匹配目标零件编号，提取对应图形实体
6. 合并匹配实体，导出新 DXF
7. ODAFileConverter: DXF → DWG（转回）
8. 保存到 uploads/part_drawings/{模具号}/{零件号}_v{版本号}.dwg
9. 旧版本 is_latest 置 false，新版本写入 entrust_drawings（version+1, is_latest=true）
```

## 六、后端接口设计

### 6.1 图纸相关接口（drawing_controller.py）

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | `/entrust/drawing/list` | 图纸列表（分页、按模具号/零件号搜索） |
| GET | `/entrust/drawing/lookup` | 批量查找图纸（询价时调用） |
| POST | `/entrust/drawing/split` | 手动触发拆图（传入模具号 + 零件号列表） |
| GET | `/entrust/drawing/preview/{mold_code}` | 预览原图中有哪些可拆的零件编号 |
| GET | `/entrust/drawing/download/{id}` | 下载子图文件 |
| DELETE | `/entrust/drawing/{id}` | 删除图纸记录和文件 |
| POST | `/entrust/drawing/upload` | 手动上传子图（跳过拆图） |

### 6.2 查找图纸接口（lookup）详细设计

**请求：**
```
GET /entrust/drawing/lookup
    ?mold_code=M250247-P6&part_codes=DIE-10,A-10,B03
```

**响应：**
```json
{
  "results": [
    {
      "part_code": "DIE-10",
      "found": true,
      "source": "database",
      "download_url": "/entrust/drawing/download/5",
      "file_path": "uploads/part_drawings/M250247-P6/DIE-10.dwg"
    },
    {
      "part_code": "A-10",
      "found": true,
      "source": "auto_split",
      "download_url": "/entrust/drawing/download/6",
      "file_path": "uploads/part_drawings/M250247-P6/A-10.dwg"
    },
    {
      "part_code": "B03",
      "found": false,
      "source": null,
      "download_url": null,
      "message": "原图中未找到该零件编号"
    }
  ]
}
```

### 6.3 现有接口改造

| 接口 | 改造内容 |
|------|---------|
| `POST /entrust/inquiry/create` | 创建询价后，自动调用 lookup 查找图纸，结果随询价数据返回 |
| `GET /entrust/inquiry/my-invitations` | 返回中增加每个零件的 `drawing_url` 字段 |
| `GET /entrust/inquiry/grouped-list` | 返回中增加图纸状态标记 |
